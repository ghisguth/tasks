Class auto_reset_event

explicit auto_reset_event( bool isset = false)
~auto_reset_event()
void set()
void wait()
bool try_wait()
Synopsis

When the auto_reset_event gets signaled, any one fiber will see this particular signal. When a fiber observes the signal by waiting on the event, it is automatically transitioned back to non-signaled state. Any fibers can subsequently set the event. 

#include <boost/fiber/spin/auto_reset_event.hpp>

class auto_reset_event : private boost::noncopyable
{
public:
    explicit auto_reset_event( bool isset = false);

    ~auto_reset_event();

    void set();

    void wait();

    bool try_wait();
};
explicit auto_reset_event( bool isset = false)

Effects:

Constructs an object of class auto_reset_event. If isset is true the variable is set.

Throws:

Nothing.

~auto_reset_event()

Precondition:

All fibers waiting on *this have been notified by a call to set (though the respective calls to wait need not have returned).

Effects:

Destroys the object.

Throws:

Nothing.

void set()

Effects:

If any fibers are currently blocked waiting on *this in a call to wait, unblocks one of those fibers.

Throws:

Nothing.

void wait()

Effects:

Blocks the current fiber. The fiber will unblock when notified by a call to this->set(). When the fiber is unblocked, the variable is reset before wait returns.

Throws:

boost::fibers::fiber_interrupted if the wait was interrupted by a call to interrupt() on the fiber object associated with the current fiber of execution.

bool try_wait()

Effects:

Returns true if the event variable is set otherwise false.

Throws:

Nothing.
