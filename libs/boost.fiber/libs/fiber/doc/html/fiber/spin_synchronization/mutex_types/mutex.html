Class mutex 

#include <boost/fibers/spin/mutex.hpp>

class mutex : private boost::noncopyable
{
public:
    mutex();
    ~mutex();

    void lock();
    bool try_lock();
    void unlock();

    typedef unique_lock< mutex > scoped_lock;
};

spin-mutex implements the lockable concept to provide an exclusive-ownership mutex. At most one fiber can own the lock on a given instance of spin-mutex at any time. Multiple concurrent calls to lock(), try_lock() and unlock() shall be permitted.

Note

spin-mutex is not bound to a scheduler and can only be used by fibers managed by different schedulers or code not running in a fiber.
