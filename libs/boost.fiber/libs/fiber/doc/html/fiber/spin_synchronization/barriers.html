Barriers

Class barrier
barrier( unsigned int initial)
bool wait()

A barrier is a simple concept. Also known as a rendezvous, it is a synchronization point between multiple contexts of execution (thread or fiber). The barrier is configured for a particular number of fibers (n), and as fibers reach the barrier they must wait until all n fibers have arrived. Once the n-th fiber has reached the barrier, all the waiting fibers can proceed, and the barrier is reset.

Note

spin-barrier is not bound to a scheduler and can be used by fibers managed by different schedulers or code not running in a fiber.
